СПЕЦИФИКАЦИЯ ДИЗАЙНА UI/UX ДЛЯ AI-ЧАТА ПРЕМИУМ-КЛАССА

ЦЕЛЬ
Создать интерфейс AI-чата премиум-уровня: спокойный, минималистичный, современный и элегантный.
Никакого визуального шума. Никакого ощущения админ-панели. Всё должно выглядеть как настоящий отполированный продукт.

ОСНОВНЫЕ ПРИНЦИПЫ
- Минимализм
- Мягкая глубина (не плоский, не текстурный)
- Полностью адаптивный layout
- Плавные анимации
- Никаких overlay-попапов, блокирующих контент

СТРУКТУРА ЭКРАНА
Интерфейс состоит из 3 зон:

1. Левая панель — чаты и пространства
2. Центр — разговор в чате
3. Правая панель — системная панель (RAG / Логи / Настройки)

Панели являются частью layout. Они сдвигают контент, но никогда не перекрывают его.

АДАПТИВНЫЙ LAYOUT
Все размеры должны быть относительными:
- Layout на процентах
- Относительные отступы
- Ограничения min/max

Интерфейс должен корректно выглядеть на маленьком ноутбуке, большом мониторе, ультрашироком экране, разделённом экране.

СТИЛЬ МЯГКОЙ ГЛУБИНЫ
Без текстур, без зернистости.

Глубина создаётся за счёт:
- тонких внутренних рамок
- мягких широких теней
- лёгких градиентов
- тонкого свечения для активных элементов

Четыре уровня глубины:
Фон -> Чат -> Панели -> Активные элементы.

ЦВЕТОВОЙ СТИЛЬ
- Тёмный графитовый фон
- Чуть более светлые панели
- Мягкий синий/фиолетовый акцент
- Никаких неоновых цветов

ПОВЕДЕНИЕ ПАНЕЛЕЙ
Левая панель:
- Выдвигается влево при закрытии
- Центр плавно расширяется

Правая панель:
- Выдвигается вправо
- Кнопки на правой стороне двигаются вместе с ней
- Панель никогда не перекрывает кнопки или чат

Режимы панели переключаются внутри контейнера:
RAG / Логи / Настройки.

АВАТАР И ВЫБОР МОДЕЛИ
Один аватар агента в верхнем левом углу чата.

Селектор модели открывается как премиум-поповер под аватаром.

Дропдаун модели должен включать:
- поле поиска
- карточки моделей с описанием
- мягкую подсветку при наведении
- тонкую анимацию

Никакого стандартного UI для дропдаунов.

RAG-ПАНЕЛЬ — ТРЕБОВАНИЯ К UI
RAG выглядит как настоящий файловый менеджер.

Структура:
- Заголовок с названием базы знаний
- Кнопки действий
- Поиск
- Дерево папок
- Превью метаданных файла
- Статистика в подвале

Поведение файловой системы:
- Drag & drop
- Переименование
- Перемещение
- Удаление (мягкое удаление)
- Множественный выбор
- Закреплённые файлы
- Подсветка активного файла

Поиск должен поддерживать:
- по имени файла
- по содержимому
- семантический поиск

СОСТОЯНИЯ UI
Панель должна поддерживать состояния:
- Пусто
- Обработка
- Готово
- Ошибка
- Устарело
- Конфликт

МИКРО-АНИМАЦИИ
- Лёгкое масштабирование при наведении
- Свечение для активных элементов
- Плавное выдвижение панели
- Плавное раскрытие папки

Никаких резких движений.

ТИПОГРАФИКА
Чистый шрифт без засечек.
Чёткая иерархия.
Читаемый на маленьких экранах.

ИКОНКИ
Минималистичные линейные иконки.
Единая толщина линий.
Без градиентов.

МОБИЛЬНЫЕ / МАЛЕНЬКИЕ ЭКРАНЫ
- Левая панель сворачиваемая
- Правая панель может стать полноэкранным листом
- Дропдаун модели центрируется

ЧЕГО НЕ ДЕЛАТЬ
- Overlay-модальные окна
- Внешний вид Bootstrap
- Зернистые текстуры
- Яркие неоновые цвета
- Плоский UI без глубины
- Фиксированные пиксельные размеры

ПРОВЕРКА КАЧЕСТВА
Дизайн готов, если:
- Выглядит хорошо на любом экране
- Панели никогда не блокируют контент
- Дропдаун модели ощущается премиальным
- RAG выглядит как настоящая файловая система
- Интерфейс ощущается спокойным и отполированным
