# Настройка системы моделей

## Автоматическое добавление моделей

Система автоматически сканирует директорию `agent-service/models` при:

1. Старте сервиса
2. Добавлении/удалении файлов моделей
3. По запросу через API `/refresh-models`

### Поддерживаемые форматы моделей
- `.bin` - бинарные файлы моделей
- `.gguf` - совместимые с GGML модели

### Логика работы
1. Система находит все файлы моделей
2. Для каждой модели определяет:
   - Имя (по имени файла)
   - Провайдер (local)
   - Версию (по дате изменения файла)
3. Добавляет или обновляет запись в базе данных

## Добавление новых моделей
Просто поместите файл модели в `agent-service/models` и система автоматически:
- Обнаружит новую модель
- Добавит её в базу данных
- Сделает доступной через API и интерфейс

## Удаление моделей
Удалите файл модели из директории, и система:
- Обнаружит отсутствие файла
- Помечает модель как неактивную в базе
- Удаляет из списка доступных
