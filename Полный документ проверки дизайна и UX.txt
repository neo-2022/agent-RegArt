AI CHAT UI QUALITY CHECKLIST
Полный документ проверки дизайна и UX

============================================================

1. ЦЕЛЬ ДОКУМЕНТА
   ============================================================

Этот чек-лист используется после реализации интерфейса.

Если все пункты выполнены — дизайн готов к продакшену.

Если нет — интерфейс дорабатывается.

============================================================
2. ОБЩЕЕ ВИЗУАЛЬНОЕ ОЩУЩЕНИЕ
============================

Проверить:

• интерфейс спокойный
• нет визуального шума
• нет зернистых текстур
• нет кислотных цветов
• нет “админского” вида

Интерфейс должен выглядеть минималистично и дорого.

============================================================
3. SOFT DEPTH ПРОВЕРКА
======================

Проверить каждый элемент:

• есть мягкая тень
• есть тонкая внутренняя граница
• есть лёгкий градиент
• активные элементы имеют subtle glow

Но:

• нет жёстких теней
• нет неонового свечения
• нет flat UI

Элементы должны быть объёмными, но аккуратными.

============================================================
4. LAYOUT ПРОВЕРКА
==================

Проверить:

• левая панель уезжает влево
• правая панель выезжает вправо
• панели не перекрывают контент
• кнопки справа двигаются вместе с панелью

Центральный чат всегда остаётся читаемым.

============================================================
5. DROPDOWN МОДЕЛЕЙ
===================

Проверить:

• dropdown открывается от аватара
• нет стандартного select
• каждая модель — карточка
• есть поиск
• выбранная модель подсвечена
• закрывается по клику вне

Должно ощущаться как premium popover.

============================================================
6. RAG ПАНЕЛЬ
=============

Проверить:

• RAG выглядит как файловая система
• папки раскрываются плавно
• есть поиск
• есть drag & drop
• есть rename/move/delete
• выбранный файл подсвечен
• есть preview

Нет таблиц и лишних рамок.

============================================================
7. UX ПОВЕДЕНИЕ
===============

Проверить сценарии:

Выбор модели
Работа с RAG
Переключение логов
Создание нового чата
Удаление чата

Во всех сценариях:

• нет скачков интерфейса
• нет неожиданных закрытий
• нет потери контекста

============================================================
8. АДАПТИВНОСТЬ
===============

Проверить на разных экранах:

• маленький ноутбук
• обычный монитор
• большой монитор
• split-screen

Проверить:

• текст читается
• панели пропорциональны
• dropdown не гигантский
• ничего не съезжает

============================================================
9. TYPOGRAPHY ПРОВЕРКА
======================

Проверить:

• читаемость текста
• правильная иерархия
• достаточный контраст
• нет декоративных шрифтов

Мелкий текст должен читаться.

============================================================
10. MOTION-ДИЗАЙН
=================

Проверить:

• hover плавный
• панели скользят мягко
• раскрытие папок плавное
• dropdown scale + fade

Нет резких движений.

Анимации короткие и аккуратные.

============================================================
11. СКОРОСТЬ И ПРОИЗВОДИТЕЛЬНОСТЬ
=================================

Проверить:

• интерфейс не лагает
• панель открывается быстро
• поиск RAG быстрый
• нет зависаний

Цель: отклик <100ms.

============================================================
12. ДОСТУПНОСТЬ
===============

Проверить:

• контраст текста достаточный
• интерфейс читаем при плохом зрении
• элементы можно нажать с тачпада
• поддержка клавиатуры

Интерфейс должен быть удобным.

============================================================
13. СОСТОЯНИЯ UI
================

Проверить состояния:

• Empty
• Loading
• Error
• Processing
• Outdated
• Conflict

Пользователь всегда понимает, что происходит.

============================================================
14. ОБРАБОТКА ОШИБОК
====================

Проверить:

• ошибки понятны
• есть кнопка retry
• интерфейс не ломается
• пользователь не теряет данные

============================================================
15. ПРОВЕРКА “УРОВЕНЬ APPLE”
============================

Задать вопросы:

• хочется ли пользоваться интерфейсом долго?
• приятно ли смотреть на экран?
• ничего ли не раздражает?
• всё ли аккуратно выровнено?
• нет ли ощущения дешёвого UI?

Если хоть один ответ “нет” — доработать.

============================================================
16. ЧЕК-ЛИСТ ДЛЯ РАЗРАБОТЧИКА
=============================

Перед сдачей проверить:

• нет overlay окон
• нет стандартных dropdown
• нет фиксированных пикселей
• нет текстур
• нет резких теней
• нет bootstrap вида

============================================================
17. ЧЕК-ЛИСТ ДЛЯ QA
===================

Проверить:

• все кнопки работают
• все панели открываются
• поиск работает
• drag & drop работает
• адаптивность работает
• ошибки обрабатываются

============================================================
18. ИТОГ

Интерфейс готов к продакшену, если:

• красив на всех экранах
• удобен
• быстрый
• аккуратный
• минималистичный
• приятный

Пользователь не хочет закрывать страницу.
